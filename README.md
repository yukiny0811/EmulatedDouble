# EmulatedDouble

`EmulatedDouble` is a software-emulated IEEE 754 binary64 value for environments such as GPU programming where native double precision is unavailable, impractical, or limited to 32-bit floating-point hardware.

All core code lives in [EmulatedDouble.h](https://github.com/yukiny0811/EmulatedDouble/tree/main/Sources/EmulatedDoubleCore/include/EmulatedDouble.h). Download or copy that single file into your project and use it from C, C++, Objective-C, Swift through Clang import, or Metal Shading Language.

Swift conversion helpers are in:

```text
Sources/EmulatedDouble/EmulatedDouble.swift
```

Copy that file too if you want the small Swift convenience extensions for converting Swift `Double` and `Float` values into `EmulatedDouble`.

## How to Use It

For C, C++, Objective-C, or Metal, include the header:

```c
#include "EmulatedDouble.h"

EmulatedDouble one = EmulatedDoubleMake(0x3ff00000u, 0u);
EmulatedDouble two = EmulatedDoubleAdd(one, one);
```

For SwiftPM users:

```swift
import EmulatedDouble

let value = 1.25.toEmulatedDouble()
let vector = EmulatedDouble3Make(value, value, value)
```

There is no separate tutorial layer. The intended reference is the code itself: read `EmulatedDouble.h`, read its `///` comments, or give the file to an AI/code assistant and ask it how to call the functions you need.

## What Is Implemented

The scalar type is `EmulatedDouble`, stored as two 32-bit words with the same bit pattern as an IEEE 754 binary64 value. The header also defines fixed-size vector and matrix helpers:

- `EmulatedDouble2`, `EmulatedDouble3`, `EmulatedDouble4`
- `EmulatedDouble2x2` through `EmulatedDouble4x4`
- SVD result types for every fixed matrix shape from `2x2` through `4x4`

Implemented scalar emulation includes bit-layout helpers, binary32 widening/narrowing, arithmetic, classification, comparisons, common functions, trigonometric functions, exponential/logarithmic functions, hyperbolic functions, rounding functions, decomposition functions, and power functions.

## Metal Notes

`EmulatedDouble` arithmetic uses software binary64 operations built on 32-bit storage and 64-bit integer math. Metal kernels using these helpers must run on targets that support the required 64-bit integer operations.

The implementation does not use native Metal `double`, host `math.h`, native `float` math, or Metal standard math functions to compute emulated math results. Fast math is not assumed; the header is written for precise/safe floating-point behavior.

## Package Contents

- `Sources/EmulatedDoubleCore/include/EmulatedDouble.h`: the single-file shared implementation to copy or download.
- `Sources/EmulatedDouble/EmulatedDouble.swift`: optional Swift convenience conversions from `Double` and `Float`.
- `Tests/EmulatedDoubleTests`: tests for bit layout, rounding, special values, scalar math, vector helpers, and matrix helpers.
